# ChangeLog
- 新增“关键词门禁”：当用户输入与助手回复同时命中同一工具的门禁关键词时，强制触发该工具调用（避免画图类需求被文字敷衍）。
- 门禁逻辑通过 AI tool 注册（每个工具独立配置 gate），并兼容既有的正则门禁（user_patterns/assistant_patterns）。
- 为 `generate_image` 工具补充关键词门禁配置，覆盖常见“画图/生成图片/出图”请求与“图片/生成图片”等回复信号。

# Risk Radar
- 串数据: No，纯逻辑变更不涉及 DB。
- 悬空引用: No，新模块仅依赖标准库。
- 幂等: No，门禁只影响是否强制工具调用，不改变工具本身幂等语义。
- 性能: No，门禁是 O(工具数×关键词数) 的短文本匹配，开销可控。
- 状态机: Yes，门禁触发会增加一次“强制工具调用”的补偿轮次（防止敷衍），可能改变对话节奏。
- 副作用: Yes，触发后会执行真实工具（如出图），相当于把“隐含需求”升级为强制副作用。
