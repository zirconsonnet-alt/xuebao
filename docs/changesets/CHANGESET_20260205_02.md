# ChangeLog
- 活动系统落库增强：`activities` 增加 `status` 字段（draft/active/ended/cancelled），并为状态+时间增加索引。
- 允许并发参与多个活动：移除 `add_participant` 的“时间重叠活动互斥”检查（仍保持同一活动只能报名一次）。
- 新增三条对外命令别名：
  - `/发起活动` → 创建活动申请（原“创建活动申请”）
  - `/通过活动 <申请ID>` → 审批活动申请并创建活动（原“审批活动”）
  - `/报名活动 <活动ID>` → 参加活动（原“我要参加”）
- 活动/申请写操作收敛到 `src/application/commands/metadata_commands.py`（通过 allow-list 防止任意字段 update）。

# Risk Radar
- 串数据: No，仅新增列与索引，默认值为 active 且回填仅覆盖已过期活动为 ended。
- 悬空引用: No，命令别名仅复用现有实现；未新增外部依赖。
- 幂等: Yes，活动审批/发起未引入幂等键（重复审批仍依赖 application.status 防重；重复报名依赖 participants PK）。
- 性能: No，新索引 `status,start_time,end_time` 有助于 active 查询。
- 状态机: Yes，存量库缺少 DB 级 status CHECK（通过命令层 allow-list 兜底）。
- 副作用: Yes，通过活动会发送群公告/消息；报名活动会写 participants 并发消息，仍可能受 NoneBot/权限影响。
